PROJECT

REVOFUN

Problem

Many small JavaScript-based games are fun and engaging, yet lack a simple platform to be showcased interactively. In addition, players and recruiters often find it difficult to view a developer’s work directly without having to download or configure the project locally.

Solution

I developed an interactive website using Vite as the bundler and JavaScript as the core game logic. This website allows users to play and explore the game directly in the browser without any additional setup. The project is designed with a Multi-Page Application (MPA) approach to clearly separate the main page and the game page while maintaining optimal performance and fast load times.

Output

The result is a personal website that showcases JavaScript-based games directly and interactively. Visitors can play the game seamlessly through a clean and responsive interface. The website is optimized for deployment on Netlify to ensure fast performance and an enjoyable user experience.

Technologies used: HTML, CSS, JavaScript, Vite

WeRent

Problem

WeRent users face several issues when renting clothes: the rental flow is too complex and time‑consuming, product quality feels unreliable, product descriptions are often inaccurate or incomplete, and it is hard to assess item condition online. There is also limited information about rental terms, fees, and policies, plus concerns about hygiene and allergies, which together reduce user trust and willingness to rent.

Solution

To address these issues, WeRent introduces a dedicated Review Page with features such as overall product rating, filtered reviews by star rating, review previews on the product detail page, and detailed customer reviews including body measurements to help assess fit. Additional features include “Read More” for long reviews, “Helpful” buttons with counts, sorting by newest or most helpful reviews, and photo/video attachments with an enlarge view, all designed to increase transparency and confidence.

Output

The expected outputs are both business and product outcomes: a fully functional review system (front‑end, back‑end, and tracking) that supports all defined user stories, plus supporting documentation, testing, and launch campaigns. In terms of metrics, WeRent targets a 30% increase in clothing rentals and a 20% increase in conversion rate by Q3 2024, and a 20% increase in retention rate and average revenue per user by Q4 2024.

Technologies used: Next.js, Node.js, Postgresql, NestJS

TicketQ

Problem

Indonesian travelers face multiple challenges when booking flights online: existing platforms have slow loading times and poor performance, interfaces are cluttered and not mobile-friendly despite most users accessing from smartphones, the lack of consolidated airline options forces users to compare prices across multiple platforms, and complex booking flows with unclear payment processes create friction. Additionally, poor search functionality and inadequate filtering options make it difficult to find suitable flights quickly, reducing user satisfaction and increasing booking abandonment rates.

Solution

To address these challenges, the project implements Hexagonal Architecture with four layers: Domain Layer containing pure business entities and port interfaces without framework dependencies, Application Layer with use cases orchestrating workflows, Infrastructure Layer with concrete adapters for TypeORM database, Midtrans payment gateway, and MMBC flight vendor API, and Presentation Layer with NestJS controllers. The architecture applies dependency inversion where use cases depend on abstractions (ports) implemented by infrastructure adapters, enabling testability and swappable implementations. Key features include rich domain models with embedded business rules, external service integration with proper authentication and error handling, and comprehensive testing across all layers.

Output

The deliverables include fully functional Transactions Service and Flight Service with complete API endpoints, proper Hexagonal Architecture implementation with clear separation of concerns, TypeScript codebase with comprehensive testing and API documentation, and successful integration with Midtrans payment gateway and MMBC flight vendor. Personal outcomes include mastery of Hexagonal Architecture and Domain-Driven Design, hands-on experience with microservices and external API integration, and proficiency in building testable, maintainable code following SOLID principles.

Technologies used:NestJS, Nx, TypeOrm, Midtrans
